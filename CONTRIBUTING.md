# Contributing to SignBridge AI

First off, thank you for considering contributing to SignBridge AI! It's people like you that make this project a great tool for the community.

## Code of Conduct

By participating in this project, you are expected to uphold our Code of Conduct:
*   Use welcoming and inclusive language.
*   Be respectful of differing viewpoints and experiences.
*   Gracefully accept constructive criticism.
*   Focus on what is best for the community.
*   Show empathy towards other community members.

## How Can I Contribute?

### Reporting Bugs

This section guides you through submitting a bug report. Following these guidelines helps maintainers and the community understand your report, reproduce the behavior, and find related reports.

*   **Check the existing issues** to see if the bug has already been reported.
*   **Use the Bug Report template** when opening a new issue.
*   **Be as detailed as possible** (OS, version, hardware specs, steps to reproduce).

### Suggesting Enhancements

If you have an idea for an enhancement:
*   Check if it has already been suggested.
*   Explain why the enhancement would be useful.
*   Provide specific examples or use cases.

### Your First Code Contribution

Unsure where to begin? Look for issues labeled `good first issue` or `help wanted`.

## Styleguides

### Git Commit Messages

*   Use the imperative mood ("Add feature" not "Added feature").
*   Limit the first line to 72 characters or less.
*   Reference issues and pull requests liberally after the first line.

### Python Styleguide

*   Follow [PEP 8](https://www.python.org/dev/peps/pep-0008/).
*   Use type hints where appropriate.

### TypeScript / React Styleguide

*   Follow standard ESLint and Prettier configurations.
*   Use functional components and hooks.
